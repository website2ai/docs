# 커스텀 도메인

전문적인 웹 프레젠스를 위해 Hando 프로젝트에 자신의 도메인을 연결하세요.

## 개요

기본적으로 웹사이트는 `yourproject.hando.dev`에서 호스팅됩니다. 커스텀 도메인을 사용하면 `www.yourdomain.com` 또는 `yourdomain.com`과 같은 자신의 도메인을 사용할 수 있습니다.

### 이점

- **전문적인 브랜딩** - 자신의 도메인 이름 사용
- **SEO 이점** - 자신의 도메인에서 권위 구축
- **기억 용이성** - 방문자가 기억하기 쉬움
- **SSL 포함** - 자동 HTTPS 인증서

### 요구 사항

- **Pro 구독** - 커스텀 도메인은 유료 플랜 필요
- **도메인 소유권** - 도메인을 소유하거나 제어해야 함
- **DNS 접근** - 레지스트라에서 DNS 레코드 수정 가능

## 설정 지침

### 1단계: 도메인 설정 열기

1. Hando에서 프로젝트 열기
2. **설정** 아이콘 (톱니바퀴) 클릭
3. **커스텀 도메인** 섹션 찾기

### 2단계: 도메인 입력

다음 형식 중 하나로 도메인 입력:

- `yourdomain.com` - 루트/apex 도메인
- `www.yourdomain.com` - WWW 서브도메인
- `app.yourdomain.com` - 커스텀 서브도메인

계속하려면 **도메인 추가** 클릭.

### 3단계: DNS 구성

도메인 레지스트라에서 서버를 가리키는 **A 레코드** 추가:

| 유형 | 이름/호스트 | 값 |
|------|-----------|-------|
| A | `@` (또는 서브도메인) | `137.66.31.203` |

**루트 도메인 (yourdomain.com):**
- 유형: `A`
- 이름/호스트: `@`
- 값: `137.66.31.203`

**서브도메인 (www.yourdomain.com):**
- 유형: `A`
- 이름/호스트: `www`
- 값: `137.66.31.203`

### 4단계: 도메인 확인

1. DNS 레코드 추가 후 Hando로 돌아가기
2. **도메인 확인** 클릭
3. DNS 확인 완료 대기

**참고:** DNS 변경은 전파되는 데 최대 48시간이 걸릴 수 있지만 대부분 1-24시간 내에 완료됩니다.

### 5단계: SSL 인증서

DNS가 확인되면:

1. SSL 인증서가 자동으로 프로비저닝됨
2. 몇 분 내에 HTTPS 활성화
3. HTTP 요청이 자동으로 HTTPS로 리디렉션

## 도메인 상태

설정에서 도메인 상태 추적:

| 상태 | 의미 | 조치 |
|--------|---------|--------|
| **대기 중** | DNS 설정 대기 중 | 레지스트라에서 A 레코드 추가 |
| **확인 중** | DNS 레코드 확인 중 | 대기 또는 확인 클릭 |
| **SSL 대기 중** | DNS 확인됨, SSL 설정 중 | 자동 SSL 대기 |
| **활성** | 도메인 활성화됨 | 도메인이 작동 중 |
| **실패** | 확인 실패 | DNS 확인 후 재시도 |

## 문제 해결

### 도메인이 확인되지 않음

**가능한 원인:**

1. **DNS가 아직 전파되지 않음** - 최대 48시간 대기
2. **잘못된 IP 주소** - `137.66.31.203` 입력 확인
3. **잘못된 레코드 유형** - A 레코드여야 함
4. **충돌하는 레코드** - 같은 이름의 다른 A 레코드 제거

**확인 방법:**

1. [dnschecker.org](https://dnschecker.org) 방문
2. 도메인 입력
3. "A" 레코드 유형 선택
4. `137.66.31.203`이 표시되는지 확인

### SSL 인증서 문제

**가능한 원인:**

1. **DNS가 완전히 전파되지 않음** - SSL은 DNS가 작동해야 함
2. **도메인 확인 미완료** - 상태가 "활성"인지 확인
3. **CAA 레코드 차단** - 제한적인 CAA 레코드 확인

**해결책:**

1. DNS가 완전히 전파될 때까지 대기
2. 몇 시간 후 확인 재시도
3. CAA 레코드가 있다면 `letsencrypt.org`를 허용된 발급자에 추가

### 웹사이트가 로드되지 않음

**가능한 원인:**

1. **SSL 프로비저닝 중** - 상태가 활성이 된 후 몇 분 대기
2. **브라우저가 이전 DNS 캐싱** - 캐시 지우거나 시크릿 모드 사용
3. **DNS가 잘못된 IP를 가리킴** - A 레코드 값 확인

**해결책:**

1. `https://` 접두사로 접근 시도
2. 브라우저 캐시 및 쿠키 삭제
3. 다른 브라우저 또는 기기 시도
4. SSL이 완전히 활성화될 때까지 15-30분 대기

### Cloudflare 프록시 문제

Cloudflare를 사용하고 문제가 있다면:

1. Cloudflare에서 DNS 설정 이동
2. A 레코드 찾기
3. 주황색 구름을 클릭하여 회색으로 변경 (DNS만)
4. 몇 분 대기 후 확인 재시도

**이유:** Cloudflare의 프록시가 SSL 인증서 프로비저닝을 방해할 수 있습니다.

## 도메인 관리

### 커스텀 도메인 제거

1. 프로젝트 **설정** → **커스텀 도메인** 이동
2. **삭제** 또는 휴지통 아이콘 클릭
3. 제거 확인

웹사이트는 `yourproject.hando.dev`로 돌아갑니다.

### 도메인 변경

다른 도메인으로 변경하려면:

1. 현재 도메인 제거
2. 새 도메인 추가
3. 새 도메인의 DNS 레코드 업데이트
4. 확인 완료

## 제한 사항

- **프로젝트당 하나의 도메인** - 각 웹사이트는 하나의 커스텀 도메인 지원
- **Pro 필요** - Free 플랜은 커스텀 도메인 사용 불가
- **hando.dev 서브도메인 불가** - `*.hando.dev` 사용 불가

## FAQ

**DNS 전파에 얼마나 걸리나요?**

보통 1-24시간이지만 드문 경우 최대 48시간이 걸릴 수 있습니다.

**HTTPS는 자동인가요?**

예, SSL 인증서가 자동으로 프로비저닝되고 갱신됩니다. 모든 트래픽은 HTTPS로 제공됩니다.

**서브도메인을 사용할 수 있나요?**

예, 모든 서브도메인이 작동합니다 (www, app, blog 등). `137.66.31.203`을 가리키는 A 레코드를 사용하세요.

**이메일에 영향을 미치나요?**

아니요, 커스텀 도메인은 A 레코드에만 영향을 줍니다. 이메일용 MX 레코드는 변경되지 않습니다.

**플랜을 다운그레이드하면 어떻게 되나요?**

Pro에서 Free로 다운그레이드하면 커스텀 도메인이 자동으로 제거됩니다.

**여러 도메인을 사용할 수 있나요?**

현재 각 프로젝트는 하나의 커스텀 도메인을 지원합니다. 여러 도메인의 경우 별도의 프로젝트를 생성하세요.

**IPv6를 구성해야 하나요?**

아니요, IPv4 (A 레코드)로 충분합니다. IPv6는 필요하지 않습니다.
