## 개요

website2ai 빌더는 웹사이트를 만들고 반복 작업을 수행하는 곳입니다. 채팅 인터페이스와 실시간 코드 생성 및 실시간 미리보기를 결합합니다.

## 인터페이스 레이아웃

### 왼쪽 패널: 채팅 및 파일

왼쪽에는 다음이 포함됩니다:

- **채팅 인터페이스**: AI와 소통하는 곳
- **메시지 히스토리**: 모든 프롬프트와 AI 응답
- **파일 트리**: 생성된 파일 탐색
- **코드 뷰어**: 파일의 소스 코드 보기

### 오른쪽 패널: 실시간 미리보기

오른쪽에는 다음이 표시됩니다:

- **상호작용 미리보기**: 실시간으로 실행되는 웹사이트
- **반응형 컨트롤**: 데스크톱, 태블릿, 모바일 뷰 간 토글
- **새로고침 버튼**: 미리보기 새로고침

### 상단 바

- **프로젝트 제목**: 클릭하여 편집
- **버전 선택기**: 버전 간 전환
- **다운로드 버튼**: 프로젝트 내보내기
- **설정**: 프로젝트 환경 설정

    - 파일 계획 중...
    - 타입 정의 생성 중...

  </Step>

  <Step title="결과 미리보기">

  </Step>
</Steps>

### 맥락 제공

좋은 프롬프트에는 다음이 포함됩니다:

- **목적**: 웹사이트의 용도
- **주요 기능**: 필요한 주요 기능
- **스타일 선호도**: 디자인 방향
- **기술**: 필요한 경우 특정 라이브러리

<CodeGroup>

```text 좋은 예시
프리랜서 디자이너를 위한 현대적인 포트폴리오 웹사이트 만들기.
포함 사항:
- 이름과 태그라인이 있는 히어로 섹션
- 카테고리별 필터링이 있는 프로젝트 갤러리
- 스킬이 있는 소개 섹션
- 연락처 폼
여백이 많은 깔끔하고 미니멀한 디자인 사용
```

```text 더 좋은 예시
Todoist와 유사한 작업 관리 앱 만들기.
기능:
- 작업 추가/편집/삭제
- 카테고리 및 태그
- 캘린더가 있는 마감일
- 다크 모드 토글
- 순서 변경을 위한 드래그 앤 드롭

디자인: 미묘한 애니메이션이 있는 현대적이고 깔끔한 인터페이스
색상: 파란색 기본, 슬레이트 배경의 다크 모드
```

</CodeGroup>

## 변경하기

### 반복적 개선

초기 웹사이트가 생성되면 다듬을 수 있습니다:

```
"푸터에 뉴스레터 가입 폼 추가"
```

```
"히어로 섹션을 더 크게 만들고 배경 그라데이션 추가"
```

```
"기본 색상을 파란색에서 보라색으로 변경"
```

```
"부드러운 스크롤 애니메이션 추가"
```

### 주요 변경

더 큰 변경의 경우 수정할 내용을 구체적으로 설명하세요:

```
"기능 섹션의 그리드 레이아웃을 캐러셀로 교체"
```

```
"카드 대신 비교 테이블이 있는 가격 섹션 재디자인"
```

```
"사용자 통계를 보여주는 차트가 있는 대시보드 페이지 추가"
```

## 진행 상황 이해

### 생성 단계

AI가 웹사이트를 만들 때 다음 단계를 볼 수 있습니다:

<Steps>
  <Step title="계획">
    AI가 프롬프트를 분석하고 파일 구조를 계획합니다

    **표시됨**: "12개 파일 계획 중..."
  </Step>

  <Step title="타입 시스템">
    모든 컴포넌트에 대한 TypeScript 인터페이스를 생성합니다

    **표시됨**: "타입 정의 생성 중..."
  </Step>

  <Step title="파일 생성">
    각 파일을 생성합니다 (병렬로 발생)

    **표시됨**:
    - ✓ package.json
    - ✓ src/App.tsx
    - ✓ src/components/Header.tsx
    - ...
  </Step>

  <Step title="완료">
    모든 파일이 생성되고 준비됨

    **표시됨**: "✨ 생성 완료!"
  </Step>
</Steps>

### 진행 상황 표시기

- **녹색 체크마크**: 파일 완료
- **회전 아이콘**: 현재 생성 중
- **오류 아이콘**: 생성 실패 (재시도 또는 재작성)

## 생성된 코드 탐색

### 파일 트리

트리에서 파일을 클릭하여 코드를 확인하세요:

- **파란색 파일**: 현재 선택됨
- **폴더 아이콘**: 디렉토리 확장/축소
- **검색**: 이름으로 파일 찾기

### 코드 뷰어

파일을 볼 때:

- **구문 강조**: 색상으로 구분된 TypeScript/CSS/JSON
- **줄 번호**: 쉬운 참조
- **복사 버튼**: 전체 파일 또는 선택한 텍스트 복사
- **다운로드**: 개별 파일 받기

## 미리보기 컨트롤

### 반응형 테스트

기기 크기 간 토글:

- **데스크톱**: 전체 너비 (1920px)
- **태블릿**: iPad 크기 (768px)
- **모바일**: iPhone 크기 (375px)

생성된 웹사이트는 자동으로 반응형입니다!

### 미리보기 새로고침

미리보기가 업데이트되지 않는 경우:

1. **새로고침 아이콘**을 클릭
2. 몇 초 기다림
3. 변경 사항이 나타나야 함

## 일반적인 문제

### 미리보기가 업데이트되지 않음

**해결책**: 미리보기 패널의 새로고침 버튼을 클릭

### 생성이 너무 오래 걸림

**정상 시간**: 30-90초

**더 오래 걸리는 경우**: 인터넷 연결을 확인하거나 페이지 새로고침을 시도

### 코드에 오류가 있음

**해결책**: AI에게 문제를 설명:

```
"제출 버튼이 작동하지 않습니다. 수정해주세요"
```

```
"Header.tsx에 TypeScript 오류가 있습니다"
```
