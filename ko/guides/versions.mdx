## 버전이란 무엇인가요?

웹사이트를 변경할 때마다 website2ai는 새로운 **버전**을 만듭니다. 버전은 서로 다른 시점의 프로젝트 스냅샷이라고 생각하세요.

- **v1**: 초기 웹사이트
- **v2**: 다크 모드 추가 후
- **v3**: 색상 변경 후
- **v4**: 애니메이션 추가 후

각 버전은 완전하고 독립적입니다 - 언제든지 이전 버전으로 돌아갈 수 있습니다.

## 왜 버전을 사용하나요?

### 안전하게 실험

작업을 잃지 않고 다양한 접근 방식을 시도하세요:

```
현재 버전 (v3): 깔끔하고 미니멀한 디자인

시도: "그라데이션으로 더 화려하게 만들기"
→ v4 생성

마음에 들지 않나요? v3로 돌아가세요!
```

### 진행 상황 추적

웹사이트가 어떻게 진화했는지 확인:

1. v1: 기본 구조
2. v2: 기능 섹션 추가
3. v3: 가격 추가
4. v4: 최종 다듬기

### A/B 테스트

동일한 기능의 다양한 버전 시도:

```
v3: 비디오 배경이 있는 히어로
v4: 이미지 슬라이더가 있는 히어로 (v2에서 분기)
v5: 애니메이션 텍스트가 있는 히어로 (v2에서 분기)

세 가지를 모두 비교하고 최고를 선택하세요!
```

## 버전 보기

### 빌더에서

상단 바의 **버전 드롭다운**을 클릭:

- 번호가 있는 모든 버전 보기
- 각각은 생성한 프롬프트를 보여줌
- 현재 버전이 강조 표시됨

### 버전 목록

```
✓ v4 - 부드러운 스크롤 애니메이션 추가 (현재)
  v3 - 색상을 파란색 테마로 변경
  v2 - 다크 모드 토글 추가
  v1 - 랜딩 페이지 만들기...
```

버전을 클릭하여 전환하세요.

## 버전 만들기

### 선형 히스토리 (기본)

변경할 때 website2ai는 다음 버전을 만듭니다:

```
v1 → v2 → v3 → v4
```

**예시:**

- 시작: v1 (기본 랜딩 페이지)
- 채팅: "가격 섹션 추가" → v2 생성
- 채팅: "후기 추가" → v3 생성
- 채팅: "더 어둡게" → v4 생성

각 버전은 이전 버전을 기반으로 합니다.

### 브랜칭 (고급)

다양한 접근 방식을 시도하고 싶으신가요? 이전 버전에서 분기하세요:

```
        ┌─ v3 (파란색 테마 시도)
v1 ─ v2 ┤
        └─ v4 (다크 테마 시도)
```

**브랜칭 방법:**

1. 분기하려는 버전으로 전환 (예: v2)
2. 변경: "파란색 색상 구성표 시도"
3. 최신 버전이 아닌 v2를 기반으로 v3 생성

## 버전 비교

website2ai에는 내장된 시각적 diff 도구가 없지만 다음을 할 수 있습니다:

1. **두 개의 탭 열기**: 서로 다른 버전을 나란히 보기
2. **둘 다 다운로드**: 로컬에서 코드 비교
3. **파일 트리 확인**: 어떤 파일이 변경되었는지 확인

  </Step>


  </Step>

  <Step title="v4: 다듬기">

    ```
    애니메이션 추가 및 간격 개선
    ```

<Steps>
  <Step title="기본 만들기 (v1)">
    ```

    ```
  </Step>

  <Step title="옵션 A 시도 (v2)">


    v2와 v3를 미리보고 최고를 선택한 다음 거기서 계속
  </Step>
</Steps>

## 버전 번호 이해

### 자동 증가

버전 번호는 자동으로 증가합니다:

- 첫 번째 생성: **v1**
- 두 번째 생성: **v2**
- 세 번째 생성: **v3**

번호를 선택하지 않습니다 - website2ai가 처리합니다!

### 비선형 번호

분기하는 경우 번호가 순차적이지 않을 수 있습니다:

```
v1 → v2 → v5 (최신)
       ↘ v3 (브랜치)
       ↘ v4 (브랜치)
```

이것은 정상입니다! 번호는 선형 히스토리가 아닌 **생성 순서**를 보여줍니다.

## 파워 유저를 위한 팁

### 빠른 비교

두 버전을 빠르게 비교하고 싶으신가요?

1. 두 개의 브라우저 창에서 website2ai 열기
2. 하나는 v3, 다른 하나는 v5로 전환
3. 나란히 비교

### 버전 태그 (해결 방법)

버전 이름을 지정할 수 없으므로 이 트릭을 사용하세요:

```
v1: "랜딩 페이지 만들기"
v2: "가격 추가 - 마일스톤: 검토 준비"
v3: "파란색 시도 - 실험"
v4: "최종 디자인 - 운영"
```

중요한 버전을 표시하려면 프롬프트에 키워드를 사용하세요!

### 롤백 전략

1. **실험 전**: 현재 버전 번호 기록
2. **변경**: 새 버전 생성
3. **마음에 들지 않나요**: 기록한 버전으로 전환
4. **계속**: 안정적인 버전에서 새 변경

## 실제 예시

실제 프로젝트가 어떻게 진화하는지 확인하세요:

```
v1: "이메일 도구를 위한 SaaS 랜딩 페이지 만들기"
    → 기본 히어로 + 기능 + CTA

v2: "3개 등급이 있는 가격 섹션 추가"
    → 가격 추가됨

v3: "고객 후기 캐러셀 추가"
    → 후기 추가됨

v4: "그라데이션으로 디자인을 더 현대적으로"
    → 시각적 새로고침

v5: "다른 히어로 레이아웃 시도" (v2에서 분기)
    → 대체 디자인

v6: "FAQ 섹션 추가" (v4에서 계속)
    → 최종 버전

결과: v6가 운영 준비 완료!
```
