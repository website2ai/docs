## Welcome to the API

The website2ai API allows you to integrate AI-powered website generation into your own applications. Build website creation features into your product, automate website generation, or create custom workflows.

<Note>
The API is designed for **developers** who want to integrate website2ai into their applications. If you just want to create websites, use the [web builder](/quickstart) instead!
</Note>

## Use Cases

- **SaaS Products**: Add AI website generation to your platform
- **Automation**: Generate websites programmatically
- **Custom Workflows**: Build tailored generation pipelines
- **Batch Processing**: Create multiple websites at once
- **Integration**: Connect website2ai to your existing tools

## Base URL

```
https://api.website2ai.com
```

## Authentication

All API requests require authentication using a JWT bearer token:

```bash
curl -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  http://localhost:8000/v1/websites
```

### Getting a Token

Authenticate using Google OAuth:

```bash
# Exchange Google OAuth code for JWT token
POST /v1/auth/google/token
Content-Type: application/json

{
  "code": "GOOGLE_AUTH_CODE"
}
```

**Response:**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
```

Store this token and include it in all subsequent requests.

## Core Concepts

### Projects and Versions

website2ai uses a **versioning system** where:

- **base_id**: Unique identifier for a project (all versions share this)
- **version**: Integer version number (starts at 1)
- Each generation creates a new version
- You can branch from any previous version

### Generation Flow

1. **Create Project**: `POST /v1/websites` - Creates base project with version 1
2. **Generate**: `POST /v1/websites/{base_id}/generate` - Generates website code
3. **Stream Progress**: `GET /sse/websites/{website_id}` - Watch real-time updates
4. **Retrieve**: `GET /v1/websites/{base_id}` - Get generated files

### Website States

- `pending`: Created but not yet generated
- `processing`: Currently generating
- `completed`: Successfully generated
- `failed`: Generation failed (check error message)

## Real-time Updates (SSE)

Subscribe to Server-Sent Events to receive file-by-file progress:

```javascript
const eventSource = new EventSource(
  `http://localhost:8001/sse/websites/${website_id}`
);

eventSource.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log(data.type, data.data);
};
```

### Event Types

| Event | Description |
|-------|-------------|
| `orchestration_started` | Generation begins |
| `planning` | Planning project structure |
| `planning_completed` | File list ready |
| `interface_planning` | Creating TypeScript interfaces |
| `interface_planning_completed` | Type system ready |
| `file_generating` | Generating specific file |
| `file_completed` | File generation complete |
| `finalizing` | Aggregating metadata |
| `completed` | All done! |
| `failed` | Generation failed |

## Response Format

All REST API responses use JSON:

```json
{
  "base_id": "uuid-of-project",
  "version": 1,
  "status": "completed",
  "title": "My Website",
  "created_at": "2025-10-21T00:00:00Z",
  "updated_at": "2025-10-21T00:05:00Z"
}
```

## Error Handling

Errors return appropriate HTTP status codes with details:

```json
{
  "detail": "Error message describing what went wrong"
}
```

### Common Status Codes

- `200 OK` - Request successful
- `201 Created` - Resource created successfully
- `400 Bad Request` - Invalid request parameters
- `401 Unauthorized` - Missing or invalid authentication
- `404 Not Found` - Resource doesn't exist
- `422 Unprocessable Entity` - Validation error
- `500 Internal Server Error` - Server error

## Rate Limits

Currently no rate limits are enforced. When deployed to production, consider implementing rate limiting based on your infrastructure needs.

## Pagination

List endpoints support pagination:

```bash
GET /v1/websites?skip=0&limit=20
```
