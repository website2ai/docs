## What Are Versions?

Every time you make a change to your website, website2ai creates a new **version**. Think of versions as snapshots of your project at different points in time.

- **v1**: Your initial website
- **v2**: After adding dark mode
- **v3**: After changing colors
- **v4**: After adding animations

Each version is complete and independent - you can always go back to any previous version.

## Understanding Version Numbers

### Auto-Incrementing

Version numbers increment automatically:

- First generation: **v1**
- Second generation: **v2**
- Third generation: **v3**

You don't choose the number - website2ai handles it!

### Non-Linear Numbers

If you branch, numbers might not be sequential:

```
v1 â†’ v2 â†’ v5 (latest)
       â†˜ v3 (branch)
       â†˜ v4 (branch)
```

This is normal! The number shows **order created**, not linear history.

## Common Questions

<AccordionGroup>
  <Accordion title="Can I delete a version?">
    Not yet, but this feature is coming soon!

    For now, all versions are saved. You can ignore ones you don't need.
  </Accordion>

  <Accordion title="Do versions count against storage?">
    Each version stores complete code, but storage is unlimited on all plans.
  </Accordion>

  <Accordion title="Can I merge versions?">
    No automatic merging yet.

    **Workaround**: Describe what you want:

    "Combine the hero section from v2 with the features from v3"
  </Accordion>

  <Accordion title="How do I know which version is deployed?">
    Versions are for development only. When you download, you choose which version to deploy.
  </Accordion>

  <Accordion title="Can I name versions?">
    Not currently - versions use numbers only. Use descriptive prompts so you remember what each version contains!
  </Accordion>
</AccordionGroup>

## Tips for Power Users

### Quick Comparison

Want to compare two versions quickly?

1. Open website2ai in two browser windows
2. Switch to v3 in one, v5 in the other
3. Compare side-by-side

### Version Tags (Workaround)

Since you can't name versions, use this trick:

```
v1: "Create landing page"
v2: "Add pricing - MILESTONE: Ready for review"
v3: "Try blue colors - EXPERIMENT"
v4: "Final design - PRODUCTION"
```

Use keywords in prompts to mark important versions!

### Rollback Strategy

1. **Before experiments**: Note current version number
2. **Make changes**: Creates new versions
3. **Don't like it**: Switch back to noted version
4. **Continue**: Make new changes from stable version
