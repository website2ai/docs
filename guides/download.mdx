## Running Locally

### Prerequisites

- **Node.js 18+** ([Download here](https://nodejs.org/))
- **npm** (comes with Node.js)

### Setup & Run

<Steps>
  <Step title="Extract ZIP">
    Unzip the downloaded file to your desired location
  </Step>

  <Step title="Open Terminal">
    Navigate to the extracted folder:

    ```bash
    cd path/to/your-website
    ```
  </Step>

  <Step title="Install Dependencies">
    ```bash
    npm install
    ```

    This installs all required packages (React, TypeScript, Tailwind CSS, etc.)
  </Step>

  <Step title="Start Dev Server">
    ```bash
    npm run dev
    ```

    Your website opens at `http://localhost:5173`
  </Step>
</Steps>

<Tip>
**Development server features:**
- Hot reload (changes appear instantly)
- TypeScript error checking
- Fast refresh
</Tip>

## Deployment Options

### Vercel (Recommended)

**Best for**: Most React apps, automatic deployments

<Steps>
  <Step title="Create Vercel Account">
    Sign up at [vercel.com](https://vercel.com) (free)
  </Step>

  <Step title="Install Vercel CLI">
    ```bash
    npm i -g vercel
    ```
  </Step>

  <Step title="Deploy">
    From your project folder:

    ```bash
    vercel
    ```

    Follow the prompts, and you're live!
  </Step>
</Steps>

**Your site will be at**: `https://your-project.vercel.app`

<Accordion title="Vercel Benefits">
  - Free tier with generous limits
  - Automatic HTTPS
  - Global CDN
  - Auto-deploy from Git
  - Preview deployments
  - Custom domains
</Accordion>

### Netlify

**Best for**: Simple deploys, form handling

<Steps>
  <Step title="Build Project">
    ```bash
    npm run build
    ```

    Creates a `dist/` folder with production files
  </Step>

  <Step title="Drag & Drop Deploy">
    1. Go to [app.netlify.com/drop](https://app.netlify.com/drop)
    2. Drag your `dist/` folder
    3. Done!
  </Step>
</Steps>

### GitHub Pages

**Best for**: Personal projects, portfolios

<Steps>
  <Step title="Install gh-pages">
    ```bash
    npm install --save-dev gh-pages
    ```
  </Step>

  <Step title="Update package.json">
    Add to `package.json`:

    ```json
    {
      "homepage": "https://yourusername.github.io/repo-name",
      "scripts": {
        "predeploy": "npm run build",
        "deploy": "gh-pages -d dist"
      }
    }
    ```
  </Step>

  <Step title="Deploy">
    ```bash
    npm run deploy
    ```

    Your site is live at the homepage URL!
  </Step>
</Steps>

### Cloudflare Pages

**Best for**: Fast global distribution

<Steps>
  <Step title="Create Account">
    Sign up at [pages.cloudflare.com](https://pages.cloudflare.com)
  </Step>

  <Step title="Connect Git">
    Connect your GitHub/GitLab repo
  </Step>

  <Step title="Configure Build">
    - Build command: `npm run build`
    - Build output: `dist`
  </Step>

  <Step title="Deploy">
    Cloudflare automatically builds and deploys!
  </Step>
</Steps>

Your project uses **Tailwind CSS**:

```tsx

  }

```

## Environment Variables

### Adding Environment Variables

Create `.env` file in project root:

```bash
# .env
VITE_API_URL=https://api.yourapp.com
VITE_API_KEY=your-api-key-here
```

Use in code:

```typescript
const apiUrl = import.meta.env.VITE_API_URL;
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="npm install fails">
    **Solution**:
    - Update Node.js to latest version
    - Clear npm cache: `npm cache clean --force`
    - Delete `node_modules` and retry
  </Accordion>

  <Accordion title="Build errors">
    **Common fixes**:
    - Check TypeScript errors: `npm run type-check`
    - Update dependencies: `npm update`
    - Check console for specific errors
  </Accordion>

  <Accordion title="Deployment fails">
    **Checklist**:
    - Build works locally? `npm run build`
    - Environment variables set?
    - Build command correct in platform settings?
  </Accordion>

  <Accordion title="Site looks different deployed">
    **Possible causes**:
    - Check environment variables
    - Verify build output includes all assets
    - Check browser console for errors
  </Accordion>
</AccordionGroup>

## Continuous Deployment

### Git + Vercel

<Steps>
  <Step title="Push to GitHub">
    ```bash
    git init
    git add .
    git commit -m "Initial commit"
    git branch -M main
    git remote add origin https://github.com/username/repo.git
    git push -u origin main
    ```
  </Step>

  <Step title="Import to Vercel">
    - Go to vercel.com
    - Click "New Project"
    - Import your GitHub repo
  </Step>

  <Step title="Auto-Deploy">
    Every `git push` automatically deploys!
  </Step>
</Steps>

### Git + Netlify

Same process as Vercel - connect repo and Netlify builds on every push!
